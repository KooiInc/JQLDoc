(inner) changeCssStyleRule(selector, rules, customCssId)

Source:
  • Styling.js, line 84
Change or create some css rule in an existing or dynamically created stylesheet (id: cssId) in the document

A @media rule can also be inserted. Use a media rule for the selector (e.g.@media print) and set the style rules for it for one or more selectors using an object with one key: mediaSelectors, e.g.:
{mediaSelectors: { "div#x": {display: `none`}, "div#y": color: `green`} }
Example
// assume changeRuleset is imported as setStyleRule
setStyleRule(".myClass", {color: "#c0c0c0", padding: "0 4px 0 15px"}, "YesItsMyCss");
//           ^ someRule   ^                                            ^
//                        ^ css rules                                  ^
//                                                                     ^ id of the stylesheet
//
// set a @media rule
setStyleRule( "@media(max-width: 1200px)",
//             ^ @media rule
              { mediaSelectors: {".myClass": {maxWidth: `75vw`}, "#someDiv": {color: red}} } )
//                                ^ selector  ^
//                                            ^ css rule(s)
Parameters:
Name Type Description
selector string the (css) selectorText, like ul li.inActive, .someClass etc.
rules Object an object with CSSStyleDeclarations
Note: enclose a string value of `content` in quotes (e.g. {content: `'Some string'`})
Note: rule keys should be valid (e.g. {marginRight: `0.3rem`} or {"margin-right": `0.3rem`})
customCssId string | undefined id of the css stylesheet (to create or retrieve), default "customCSS"
